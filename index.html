Vision

A home for an AI that answers to you.

Most AI products ask you to trust a black box. Toby takes the opposite route: a private agent you can run yourself, or host with boundaries you can explain.

The point is simple. Toby should help with real work without leaving you to wonder where data went, which tools it touched, or who approved the run.

Security posture

Built for people who want boundaries they can name.

Toby is built around least privilege, reviewable activity, and controls that match the work it is allowed to do.

Data boundaries

Per-workspace storage, retention controls, and separation between customer data and service telemetry.

Key custody

Encryption at rest by default, with BYOK and customer KMS patterns for dedicated and owned cloud deployments.

Runtime isolation

Agent workers use scoped secrets, limited tool grants, and isolation that matches the deployment.

Connector consent

Tool access is explicit and revocable. Each connector should get only the permission it needs.

Audit history

You can review task runs, tool calls, approvals, connector changes, and admin actions.

Network control

Egress allowlists, private networking, and environment controls are available for dedicated and owned cloud deployments.

Privacy defaults

Fewer promises. Clearer defaults.

  • 01 Toby-owned models do not train on customer content by default.
  • 02 Sensitive connectors should be scoped, time limited, and easy to revoke.
  • 03 Risky external actions can require a person to approve them first.
  • 04 The system should show what happened, which tools were used, and why.
